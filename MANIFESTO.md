# The Four Principles of Earned Autonomy

> Most AI tools forget. Some take notes. Almost none verify.
> Almost none earn the right to act.

After enough sessions, your agent's memory becomes a graveyard of
half-true facts and aspirational rules. Skills it claims to have
master are skills it has never been tested on. Patterns it
"learned" are patterns it stumbled into once and recorded as truth.

compounded exists because compounding only works if what compounds is
correct, and trust only works if it is earned.

## 1. Bounded over Unbounded

Memory should be small enough to read in one breath.

Force consolidation, not accumulation. The brain is a filter,
not a hard drive. A 200-line memory file is a maintenance
problem; a 30-line one is a tool.

## 2. Verified over Recalled

A skill the agent saved is a hypothesis.
A skill that replayed successfully on a fresh task is a fact.

The difference is the next bug you avoid. Notes are cheap.
Verification is the price of trust.

## 3. Earned over Granted

Authority should be earned, never assumed.

A new skill runs only when invoked. After three clean uses,
it can offer to run when relevant. After ten clean uses, it
runs without asking. One correction sends it back a step.

This is how you train a junior. It is how you should train
your agent.

## 4. Composable over Comprehensive

compounded does three things: cross-project memory, the trust
gradient, portable archive.

It does not capture every observation. It does not consolidate
your project notes. It does not run a daemon or call home.

AutoMemory writes. AutoDream cleans. Skill-creator evaluates.
compounded earns.

A small tool that does one thing right is worth a hundred
tools that do everything ambivalently.
